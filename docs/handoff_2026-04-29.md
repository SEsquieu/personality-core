## Useful Commands

Install dev dependencies:

```powershell
pip install -e ".[dev]"
```

Run tests:

```powershell
$env:PYTHONPATH='src'; python -m pytest
```

Compile check:

```powershell
python -m compileall -q src tests
```

Try one personality stack:

```powershell
personality-core chat "Explain why retries hide real errors." --model "ollama/gemma4:e4b" --max-tokens 300 --debug
```



## Known Behavior

### Gemma 4 and empty output

Gemma 4 via Ollama can return empty visible content when thinking consumes the token budget. Personality Core now sends `think: false` by default to avoid that.

If testing thinking mode intentionally, use a larger token cap:

```powershell
personality-core chat "Explain why retries hide real errors." --model "ollama/gemma4:e4b" --max-tokens 500 --think --debug
```

### Truncated output

If output cuts off mid-sentence or mid-list, check:

```json
"model_response": {
  "done_reason": "length"
}
```

Increase `--max-tokens`.

### Sarcasm scoring

The current heuristic evaluator often reports:

```text
Sarcasm Core target appears under-expressed.
```

This is expected for now. The heuristic scorer only checks a small set of sarcasm marker phrases. It is useful scaffolding, not a real style judge yet.

## Good Next Steps

### 1. Improve evaluator output

Add per-core scores:

```json
{
  "core_scores": {
    "technical_core": 0.88,
    "sarcasm_core": 0.31,
    "low_verbosity_core": 0.79
  }
}
```

This will make `compare` much more actionable.

### 2. Make stabilizer demos visible

When `--stabilizer` runs, show:

- raw output
- repair reason
- repaired output
- before/after score

Right now repair happens, but it is not showcased well.

### 3. Tune the compiler for style expression

Gemma 4 is still conservative with sarcasm. Consider stronger compiled instructions when sarcasm target is high, while preserving boundaries.

Possible change:

```text
Include one brief dry aside when the user asks about an obviously flawed technical pattern.
```

### 4. Add OpenAI adapter

The pipeline already routes non-`ollama/` models to `OpenAIAdapter`, but that adapter is still a placeholder.

Public usability improves a lot if this works:

```powershell
$env:OPENAI_API_KEY='...'
personality-core chat "Explain retry loops." --model "gpt-4.1-mini"
```

### 5. Future frontend

A good first frontend should not be a marketing page. It should be the actual tool:

- prompt input
- model selector
- installed core stack
- sliders
- compare tabs
- output panel
- core debug/trace panel
- drift/stabilizer panel
